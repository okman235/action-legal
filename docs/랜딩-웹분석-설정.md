# 랜딩 페이지 웹 분석(유입경로) 설정 가이드

랜딩 페이지 파일: **`landing/index.html`**
공개 URL(예): `https://okman235.github.io/action-legal/landing/`
> 개인정보처리방침(`index.html`)은 앱 심사용이라 URL을 그대로 두고, 마케팅 랜딩은 `/landing/` 경로로 분리했습니다.

---

## 1. 무엇을 추적하나

- **유입 소스 캡처**: 방문 URL의 `utm_*` 파라미터 + `document.referrer`를 읽어 유입 소스를 판별합니다.
  - UTM이 없어도 리퍼러 도메인으로 소스를 추정합니다 (instagram / tiktok / youtube / naver / google …).
- **first-touch / last-touch 저장**: 최초 유입과 최근 유입을 `localStorage`에 저장합니다.
- **GA4 전송**(선택): 측정 ID를 넣으면 `traffic_source`, `app_store_click` 이벤트를 GA4로 보냅니다.
- **웹 → App Store 연결**: 다운로드 버튼 클릭 시 유입 소스를 **App Store 캠페인 토큰(`ct`)** 으로 붙여, App Store Connect 앱 분석의 "캠페인"에 잡히게 합니다.

---

## 2. 설정 (파일 상단 3가지만 채우면 끝)

`landing/index.html`의 `window.ACTION_ANALYTICS` 블록을 수정합니다.

```js
window.ACTION_ANALYTICS = {
  GA4_MEASUREMENT_ID: '',                          // ① GA4 측정 ID (예: 'G-XXXXXXXXXX')
  APP_STORE_URL: 'https://apps.apple.com/kr/app/', // ② 실제 App Store 앱 링크
  APPEND_CAMPAIGN_TOKEN: true                      // ③ 캠페인 토큰 자동 부착 여부
};
```

### ① GA4 측정 ID 얻기
1. [Google Analytics](https://analytics.google.com) → 관리 → 데이터 스트림 → 웹 스트림 생성
2. 발급되는 `G-XXXXXXXXXX` 를 `GA4_MEASUREMENT_ID`에 입력
3. 비워두면 GA 전송만 건너뛰고, 자체 유입 캡처는 그대로 동작합니다.

### ② App Store 링크
- 앱 출시 후 실제 링크(`https://apps.apple.com/kr/app/id0000000000`)로 교체하세요.

## 3. 채널별 캠페인 링크 만들기 (중요)

"어디서 왔는지"를 구분하려면 **채널마다 다른 UTM 링크**를 배포해야 합니다.

형식:
```
https://okman235.github.io/action-legal/landing/?utm_source=<채널>&utm_medium=<유형>&utm_campaign=<캠페인명>
```

예시:

| 배포 위치 | 링크 |
|---|---|
| 인스타 프로필 | `...?utm_source=instagram&utm_medium=bio&utm_campaign=profile` |
| 틱톡 게시물 | `...?utm_source=tiktok&utm_medium=video&utm_campaign=acting_reaction` |
| 유튜브 설명란 | `...?utm_source=youtube&utm_medium=desc&utm_campaign=diction_tip` |
| 네이버 카페 후기 | `...?utm_source=naver&utm_medium=cafe&utm_campaign=review` |
| 인플루언서 A | `...?utm_source=influencer&utm_medium=paid&utm_campaign=creatorA` |

이렇게 하면 GA4 리포트와 App Store 캠페인 데이터에서 채널별로 분리되어 보입니다.

---

## 4. 데이터 확인 방법

- **GA4**: 보고서 → 획득 → 트래픽 획득에서 소스/매체별 방문·이벤트 확인. `app_store_click` 이벤트로 다운로드 버튼 클릭률 파악.
- **App Store Connect → 앱 분석**: "소스 유형"과 "캠페인"에서 웹 유입·캠페인 토큰별 다운로드·전환 확인.
- **브라우저 콘솔**: 페이지에서 `window.__actionAttribution` 로 현재 방문의 캡처된 유입 정보를 즉시 확인할 수 있습니다.

---

## 5. 퍼널로 이어 보기

```
채널 링크 노출 → 랜딩 방문(GA) → 다운로드 클릭(app_store_click) → App Store 설치(앱 분석) → 무료체험 → 유료 전환
```

채널별로 이 퍼널의 전환율과 **CAC(획득 비용)** 를 비교해 예산을 재배분하세요.
(자세한 채널 전략은 [`유입경로분석.md`](./유입경로분석.md) 참고)

---

## 6. 개인정보 관련 참고

- GA4는 쿠키를 사용합니다. 랜딩 푸터에 관련 고지 문구를 넣어 두었고, GA 설정에는 `anonymize_ip`를 적용했습니다.
- 앱 개인정보처리방침(`index.html`)은 **앱** 데이터 처리에 관한 것으로, 이 웹 분석과는 별개입니다. 웹 쿠키 사용을 방침에 명시하고 싶다면 별도 문구 추가를 검토하세요.
